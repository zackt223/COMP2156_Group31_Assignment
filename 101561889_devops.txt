COMP2156 - DevOps for System Administration teaches students
how to bridge the gap between software development and IT operations.
The course focuses on automation, collaboration, and continuous delivery.

Tools covered in COMP2156 include:
- Git and GitHub for version control
- Docker for containerization
- Jenkins/GitHub Actions for CI/CD pipelines
- Linux for system administration

COMP2156 assessments include:
- Group assignments focused on real-world DevOps workflows
- Individual labs on Linux and scripting
- A final project integrating all DevOps concepts learned

COMP2156 - DevOps for System Administration is a course at George Brown College
that introduces students to the culture, practices, and tools of DevOps.
The course focuses on breaking down barriers between development and operations teams
to deliver software faster, more reliably, and at scale.

Core tools and technologies taught in COMP2156:
- Git and GitHub for distributed version control
- Linux command line for system administration tasks
- Docker for building and managing containers
- GitHub Actions/Travis CI for automating CI/CD workflows
- Bash scripting for task automation
